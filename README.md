## Reflection

### How can I ensure that my code, program, or software is functional and secure?

I can ensure my code is functional by testing it against the requirements and making sure each feature works the way it is supposed to. For the Contact Service project, I used JUnit tests to check that contact IDs were unique, required fields were not null, and fields followed the correct length or format rules. Testing helped me catch possible errors before the program would be used by someone else.

To help keep software secure, I need to validate user input, avoid accepting invalid data, and write code that prevents errors instead of ignoring them. Good testing also supports security because it helps make sure the program handles incorrect or unexpected input safely.

### How do I interpret user needs and incorporate them into a program?

I interpret user needs by carefully reviewing the project requirements and turning them into specific features and rules in the program. For example, the Contact Service requirements explained what information a contact needed and what limits each field had. I used those requirements to decide how the Contact class and ContactService class should work.

I also incorporated user needs by creating tests that matched the requirements. This helped confirm that the program did what the user asked for, not just what I assumed it should do.

### How do I approach designing software?

My approach to designing software is to first understand the requirements and break the program into smaller parts. I think about what each class should be responsible for and how the pieces should work together. In this project, the Contact class stored contact information, while the ContactService class handled adding, deleting, and updating contacts.

I also try to design software with testing in mind. Writing unit tests helps me confirm that each part of the program works correctly on its own. This makes the code easier to maintain, easier to fix, and more reliable overall.
